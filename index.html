Veebilehe loomine, privaatsed veebilehitsejad

Käesoleva töö eesmärk on õppida praktiliselt veebiarendust, luua ise toimiv veebileht ning uurida, kuidas veebilehitsejad kasutaja privaatsust kaitsevad, eriti seoses browser fingerprintinguga¹. Veebiarendus on üks olulisemaid tehnoloogiaid, mis toetab kaasaegset suhtlust. Kõik arvutite ja Internetiga seotud tehnoloogiad mind huvitavad. Loovtöö on hea võimalus tekitada kasulikku kogemust... Plaanis on läbida Edube platvormil kursuseid, hostida veebisait Verceli abil ning kasutada avalikku tasuta koodi veebilehitsejate fingerprintingu vastaste tehnoloogiate testimiseks.

¹ Browser fingerprinting — veebilehitseja ning arvuti omaduste, käitumise põhjal kasutaja äratundmine.

MÕISTED

Siin on tööga seotud mõisted ja seletused.

Veebi infrastruktuur

Veebilehitseja (brauser)

Veebilehitseja on programm, mis võimaldab kasutajal avada ja vaadata veebilehti. See tõlgendab HTML-, CSS- ja JavaScript-koodi ning kuvab selle kasutajale arusaadaval kujul. Tuntuimad brauserid on Google Chrome, Mozilla Firefox ning Safari.

Veebileht, veebisait

Veebileht on dokument, mis on loodud veebitehnoloogiate abil ning mida brauser kasutajale kuvab. See võib sisaldada teksti, pilte, linke, videoid, nuppe jne.

Veebilehed on omavahel lingitud ning nende kogum moodustab veebisaidi. Veebisait võib koosneda ühest või mitmest veebilehest, mis moodustavad ühtse terviku.

Veebisaidid on näiteks google.com ja youtube.com. Vaadates YouTube’is videot, vaatad ühte YouTube’i veebilehte, kus on videopleier, kommentaarid, kirjeldus ja soovitused.

Server, hostimine

Server on arvuti, mis talletab veebisaidi faile ja võimaldab kasutajatel neile ligi pääseda. Server vastab brauseri päringutele, saates kasutajale veebilehe jaoks vajalikud failid.

Hostimine tähendab seda, et veebisaidi failid asuvad serveris ning on Interneti kaudu kättesaadavad.

Veebiarenduse põhitehnoloogiad

HTML – Hyper Text Markup Language

HTML on veebi põhistruktuuri kirjeldav märgistuskeel (markup language). See määrab kõik veebilehe elemendid ja nende omadused ning impordib CSS- ja JavaScript-failid. Brauser kasutab HTML-i, et veebileht üldse üles ehitada.

CSS – Cascading Style Sheets

CSS-iga saab määrata veebilehe ja selle elementide välimuse, näiteks värvid, paigutuse, fondid ja animatsioonid.

JavaScript (JS)

JavaScript lisab veebilehele interaktiivsuse ja dünaamilise käitumise. See võimaldab reageerida kasutaja tegevustele, laadida andmeid või muuta lehe elemente ilma lehte uuesti laadimata.

JavaScript on programmeerimiskeel ning selle kasutusvõimalused on väga laiad. Samas saab JavaScripti abil ka kasutajat rünnata, näiteks pahatahtliku koodi kaudu.

API – Application Programming Interface

API on reeglite ja funktsioonide kogum, mis võimaldab programmidel kasutada süsteemi või keskkonna võimalusi, näiteks laadida Google Mapsi või kasutada kaamerat videokõneks.

Veebilehitseja pakub mitmeid API-sid, mis võimaldavad näiteks joonistada ekraanile, töödelda heli, lugeda ekraani suurust või kasutada seadme funktsioone. API lihtsustab keerukaid protsesse, pakkudes programmeerijale selget liidest.

Tarkvara

Open-source, OSS, closed-source

Open-source tähendab, et tarkvara lähtekood on kõigile nähtav, kasutatav, muudetav ja jagatav. See võimaldab arendajatel ühiselt tarkvara täiustada ning kasutajatel mõista, kuidas programm töötab.

Closed-source tarkvara puhul ei ole lähtekood avalik ning kasutajad ei saa seda vabalt muuta ega uurida.

OSS (Open Source Software) tähendab tarkvara, mille lähtekood on open-source. Näited: Linux, Firefox, Chromium.

Git, GitHub

Git on versioonihaldussüsteem, mida kasutatakse koodi muudatuste jälgimiseks.

  • projektide varasemate versioonide salvestamine
  • mitmekesi sama projekti kallal töötamine
  • erinevate muudatuste ühendamine

GitHub on veebikeskkond, mis võimaldab Gitiga hallatavaid projekte Internetis hoiustada ja jagada. GitHubis on palju avalikku OSS-koodi, kuid alati tuleb lugeda litsentsi.

Fork

Fork tähendab projekti koopiat, mis luuakse selleks, et teha oma muudatusi originaalprojekti mõjutamata. Forkimist kasutatakse GitHubis ja teistes koodihoidlates. Näiteks LibreWolf on Firefoxi fork.

Muud mõisted

Anti-fingerprinting, ad-block

Anti-fingerprinting tähendab brauseri kaitset seadme ja kasutaja unikaalse tuvastamise (fingerprintimise) vastu.

Ad-block on reklaamiblokeerija. Selle kasutamine võib viia sisu piiramise või peitmiseni, kuigi see ei ole tavaliselt seadusega karistatav.

OS, GPU, CPU

OS (Operating System) tähendab operatsioonisüsteemi, CPU (Central Processing Unit) on protsessor ning GPU (Graphics Processing Unit) on graafikakaart.

TÖÖARENG

1. JavaScript Institute kursus

JS Institute pakub kahte põhilist kursust veebiarenduse alustamiseks: HTML Essentials ja CSS Essentials. Need kursused on mõeldud algajatele, kes soovivad õppida veebilehtede loomise aluseid, ning need valmistavad ette Web Development Essentials sertifikaadi sooritamiseks.

HTML Essentials kursus õpetab HTML-i põhistruktuuri, semantilisi elemente, linke, pilte, tabeleid, vorme ning veebilehe ülesehituse loogikat.

CSS Essentials keskendub veebilehtede kujundamisele. Kursus katab CSS süntaksi, valijad, värvid, fondid, box-mudeli, paigutussüsteemid (Flexbox, Grid), animatsioonid ja responsiivse disaini põhimõtted.

2. Veebilehe loomine

Lõin oma veebilehe, kasutades õppitud keeli. Koodi kirjutasin programmis WebStorm, mis on spetsiaalselt veebiarenduseks loodud mugav tööriist. Kujunduse ja sisu tegin ise, kasutades ka tasuta vahendeid nagu Google Fonts ja Font Awesome. Google Fontsist saab tasuta erinevaid kirjatüüpe ning Font Awesome pakub ikoone, mis parandavad saidi väljanägemist.

Veebilehe aadress: https://private-browsers.vercel.app

3. Andmete kogumine ja analüüs

GitHubis leidsin kaks tasuta kasutatavat koodibaasi, mis sobivad hästi brauserite testimiseks:

  • FingerprintJS
  • ClientJS

Genereerisin ChatGPT abil skripti, mis kasutab GitHubi koodi ja annab saidil võimaluse alla laadida JSON-faili, mis sisaldab infot, mida FingerprintJS ja ClientJS kasutaja kohta välja nühivad.

Kasutasin Pythoni skripti, et koondada JSON-failid suureks CSV-ks. Selle analüüsimisel selgus, millised brauserid kaitsevad fingerprintingu eest kõige paremini.

BRAUSERID

Testin järgnevad veebilehitsejad:

Google Chrome
See on Google’i brauser, kõige kasutatuim maailmas, loodud aastal 2008. Põhineb Chromiumil, ei oma anti-fingerprintingu tehnoloogiaid ega muid, mis kaitseks jälgimisest või reklaamidest ning see sisaldab palju Google’i telemeetriat. See ei tähenda, et Chrome ei ole turvaline, vaid et see pole privaatne.

Mozilla Firefox
Netscape’i open-source'itud koodist sündinud brauser. Mozilla on organisatsioon, tekkinud peale Netscape’i surma, mille eesmärk on arendada ja hallata Firefoxi. See on täiesti OSS ning on üks vanimatest brauseritest.

Microsoft Edge
See on põhimõtteliselt Chrome Microsofti kastmes, mis surub kasutajatele peale tooteid, mis pole just armastatud väga inimeste poolt. Peaks olema sarnase anti-fingerprintinguga ehk seda pole. Kuid võib olla reklaami ja muu jälitamisvastaseid tehnoloogiaid.

Opera
See on Chromiumil põhinev brauser, mis oli varem oma mootoriga. Privaatsuse poolest peaks olema Chrome’ist natuke tugevam, kuna on olemas ad-block.

Selle omanik on vahetunud paar korda, aga praegu omab seda Hiina firma. Omanik maksab juutuuberitele reklaami eest ning inimestele öeldakse, et see on parim brauser. Üks põhjustest on VPN. Opera omanik on üsna kahtlane ja sellega on seotud mitu skandaali. Ühesõnaga see VPN on ebaturvaline ning omanik ei pea seadustest kinni. Chrome võib olla isegi parem.

Brave
See on minu arvates ainuke enam-vähem tuntud privaatne Chromiumi põhine veebibrauser. Selle loojad on Brendan Eich ja Brian Bondy. Esimene on JavaScripti looja ning üks Mozilla asutajatest.

See brauser tuleb ad-blockiga ning jälitamisvastase kaitsega.

LibreWolf
Kogukonna loodud alternatiiv Firefoxile ilma telemeetriata ja muudetud sätetega, uBlock Origin on juba olemas. See on üks parimatest paigaldustest, mis kaitseb jälgijate, reklaamide ja kahjuliku vastu.

Mullvad
Aasta 2023. Mitte tavaline brauser. Loodud Mullvad VPN-i ja Tor Projecti koostöös, et luua tugev tööriist privaatseks veebisurfamiseks – uBlock Origin ja NoScript paigaldused on juba olemas, mõeldud Mullvad VPN-i (või muu) ja DNS-iga kasutamiseks. Blokeerib maksimaalselt ja kustutab sessiooni järel kõik jäljed (arvutis). Töötab sarnasel viisil nagu Tor – kõik kasutajad on sarnased, iga seanss on kindlad API-d muudetud, et paistaks iga kord erinev kasutaja.

TULEMUSED

Chromium – halvim üldse

Chrome ei oma sisseehitatud anti-fingerprintingut. Mitte “halb” – täiesti olematu.

Testi põhjal näeb:

  • FingerprintJS visitorId ei muutunud sessioonide vahel üldse → täielik püsiv ja identne prindimuster.
  • ClientJS fingerprint oli samuti seansside vahel sama → skriptide tasemel null kaitset.

Chrome näitab ilma kaitseta:

  • täpne ekraani resolutsioon
  • täpne OS versioon
  • kasutatud fontide komplekt
  • WebGL renderer ja GPU nimi
  • audio context fingerprint
  • canvas fingerprint täpselt sama
  • JS heap suurus, mälu, CPU tuumade arv
  • ping statistika, RTT

Miks Chrome on halb? Sest Google ei näe privaatsust äriliselt kasulikuna. Chrome on maailma suurima reklaamifirma toode. Anti-fingerprinting mõttes on see sisuliselt “avatud uksed” mõne kõige agressiivsema trackija ees.

Microsoft Edge

Edge on täielikult Chrome’i kloon, ainult:

  • natuke rangem SmartScreen
  • natuke rohkem sandboxi
  • veidi piiranguid, kui “tracking protection” on High

Test ütleb:

  • FingerprintJS VisitorId ei muutu sessioonide vahel
  • ClientJS fingerprint ei muutu sessioonide vahel → sama lihtsalt fingerprintitav kui Chrome

Edge lisab:

  • Microsofti enda telemeetriat
  • Microsofti reklaame
  • Bing integratsiooni

See teeb olukorra tegelikult halvemaks, kui soovid privaatsust.

Opera

Opera müüs end Hiina konsortsiumile ja:

  • Opera serveritega toimub pidev taustsünkroon
  • Opera VPN ei ole VPN, vaid proxy
  • Anti-fingerprintingu mehhanismi pole olemas

Sinu CSV põhjal: igas sessioonis sama fingerprint.

Opera näitab sama infot kui Chrome, lisaks:

  • seadme resolutsioon
  • fontide komplekt
  • audio fingerprint
  • canvas fingerprint
  • WebGL fingerprint
  • GPU nimi

Opera on pigem hullem kui Chrome, sest lisaks fingerprintingule on neil oma “built-in analytics”.

Chromium (üldiselt)

Chromium eksponeerib veebilehtedele:

  • täpne süsteemi fontide loend
  • GPU / WebGL signatuurid
  • brauseri ja OS-i täis User-Agent
  • ekraanisuuruse täpsed väärtused
  • kõik WebRTC andmed
  • pluginate ja codec-ite info

Chromiumi puhul ei toimu normaliseerimist ega randomiseerimist – iga väärtus, mis on sinu riistvara konfiguratsioonist saadav, jõuab trackeriteni.

Seetõttu on Chromiumi baasil brauserid anti-fingerprinting seisukohalt halvimad valikud.

Brave Browser – osaline ja minimaalne anti-fingerprinting

Brave põhineb Chromiumil, kuid lisab oma “Shields” ja teatud anti-tracking kaitsed.

Brave teeb järgmist:

  • blokib vaikimisi reklaamid ja kolmandate osapoolte trackerid
  • võib randomiseerida mõningaid canvas/WebGL signatuure
  • blokib kolmanda osapoole küpsised

Kuid Brave ei standardiseeri kõiki kasutajaid ühte anonüümsusgruppi. GPU ja fontide loend jäävad nähtavaks ning pikaajaline fingerprint säilib.

Mozilla Firefox (tavaline)

Firefox sisaldab privacy.resistFingerprinting (RFP) mehhanismi, mida saab about:config kaudu lubada.

Firefoxi kaitsed:

  • ajavööndi normaliseerimine
  • ekraani letterboxing
  • User-Agenti üldistamine
  • graafika-API vastuste moonutamine

Firefoxi tugevus sõltub konfiguratsioonist – vaikimisi on see keskpärane, kuid RFP lubamisel muutub oluliselt tugevamaks.

LibreWolf – tugev Firefoxi fork

LibreWolf on Firefoxi baasil, kuid privaatsusele karmilt häälestatud.

  • telemeetria eemaldatud
  • agressiivne trackerite blokeerimine
  • resistFingerprinting vaikimisi tugev
  • uBlock Origin kaasas

LibreWolf ei ole täiesti homogeenne nagu Tor, kuid on oluliselt parem kui Chromium-põhised brauserid.

Mullvad Browser – Tor Browser ilma Tor-võrguta

Mullvad Browser põhineb Tor Browseri anti-fingerprinting tuumal, kuid ei kasuta Tor võrku.

  • resistFingerprinting täielikult lubatud
  • User-Agent ja OS parameetrid standardiseeritud
  • piiratud fontide komplekt
  • API väärtused võivad iga seansiga muutuda
  • telemeetria täielikult eemaldatud
  • letterboxing ekraanisuurustele

Mullvad Browseri eesmärk on maksimaalne kasutajate homogeniseerimine, mis teeb pikaajalise jälgimise äärmiselt raskeks.
