Veebilehe loomine, privaatsed veebilehitsejad

Käesoleva töö eesmärk on õppida praktiliselt veebiarendust, luua ise toimiv veebileht ning uurida, kuidas veebilehitsejad kasutaja privaatsust kaitsevad, eriti seoses browser fingerprintinguga¹. Veebiarendus on üks olulisemaid tehnoloogiaid, mis toetab kaasaegset suhtlust. Kõik arvutite ja Internetiga seotud tehnoloogiad mind huvitavad. Loovtöö on hea võimalus tekitada kasulikku kogemust... Plaanis on läbida Edube platvormil kursuseid, hostida veebisait Verceli abil ning kasutada avalikku tasuta koodi veebilehitsejate fingerprintingu vastaste tehnoloogiate testimiseks.

¹ Browser fingerprinting — veebilehitseja ning arvuti omaduste, käitumise põhjal kasutaja äratundmine.

MÕISTED

Veebilehitseja (brauser)

Veebilehitseja on programm, mis võimaldab kasutajal avada ja vaadata veebilehti. See tõlgendab HTML-, CSS- ja JavaScript-koodi ning kuvab selle kasutajale visuaalselt arusaadaval kujul. Näideteks on Chrome, Firefox ja Safari.

Veebileht

Veebileht on dokument, mis on loodud veebi keeltes ja mida server edastab brauserile. See võib sisaldada teksti, pilte, linke, videoid ja interaktiivsust. Veebilehtede kogum moodustab veebisaidi.

Server

Server on arvuti või teenus, mis saadab veebilehti ja andmeid kasutajatele üle interneti. See töötleb päringuid ja tagastab vastused, näiteks HTML-failid või API andmed. Serverid töötavad pidevalt, et kasutajad saaksid sisu igal ajal kätte.

Hostimine

Hostimine on teenus, mis võimaldab veebilehtedel olla internetis kättesaadavad. Teenusepakkuja salvestab failid serveris ja tagab nende toimimise. Näiteks kasutavad inimesed hostimiseks teenuseid nagu Netlify, OVH või Zone.

Fingerprinting

Fingerprinting on tehnika, millega veebilehed koguvad kasutaja seadme ja brauseri unikaalseid tunnuseid tuvastamiseks. See ei vaja küpsiseid, sest põhineb süsteemi omadustel nagu GPU, fondid ja ekraanisuurus. Näited: Canvas fingerprinting, WebGL signatuurid, AudioContext fingerprinting.

Veebi keeled

HTML – Hyper Text Markup Language

HTML on veebi põhistruktuuri kirjeldav märgistuskeel. See määrab, millised elemendid nagu pealkirjad, lõigud, pildid ja lingid veebilehel asuvad. Brauser kasutab HTML-i, et veebileht üldse üles ehitada.

CSS – Cascading Style Sheets

CSS määrab veebilehe visuaalse välimuse, nagu värvid, paigutus ja fondid. Seda kasutatakse koos HTML-iga, et kujundada veebilehti atraktiivseks ja kasutajasõbralikuks. Näiteks saab CSS-iga muuta taustavärvi või joondada elemente.

JavaScript (JS)

JavaScript lisab veebilehele interaktiivsuse ja dünaamilise käitumise. See võimaldab reageerida kasutaja tegevustele, laadida andmeid või muuta lehe elemente ilma uuesti laadimiseta. Näiteks avanevad menüüd ja vormide kontroll töötavad sageli JavaScriptiga.

Git (GitHub)

Git on versioonihalduse süsteem, mis võimaldab arendajatel jälgida muudatusi koodis ning töötada sama projektiga samaaegselt ilma konflikte tekitamata. See võimaldab luua salvestuspunkte (commits), taastada varasemaid versioone ja hallata erinevaid arendusharusid.

GitHub on Gitil põhinev veebiplatvorm, kus saab projekte hoida pilves, teha koostööd, jagada koodi ja kasutada automatiseerimisvahendeid. GitHub lihtsustab Gitiga töötamist, pakkudes graafilist liidest, meeskonnatöö vahendeid ja projektihalduse funktsioone.

TULEMUSED

CHROMIUM– HALVIM ÜLDSE

Chromiumil ei ole sisseehitatud anti-fingerprintingut. Mitte “halb” – täiesti olematu.

Testi põhjal näeb:

  • FingerprintJS visitorId ei muutunud sessioonide vahel üldse → st. täielik püsiv ja identne prindimuster.
  • ClientJS fingerprint oli ka seansside vahel sama → st. skriptide tasemel null kaitset.

Chromium näitab ilma kaitseta:

  • täpne ekraani resolutsioon
  • täpne OS versioon
  • kasutatud fontide komplekt
  • WebGL renderer ja GPU nimi
  • audio context fingerprint
  • canvas fingerprint täpselt sama
  • JS heap suurus, mälu, CPU tuumade arv
  • ping statistika, RTT

Miks Chromium on halb?
Sest Google ei näe privaatsust äriliselt kasulikuna. Chrome on maailma suurima reklaamifirma toode. Anti-fingerprinting mõttes on see sisuliselt “avatud uksed” mõne kõige agressiivsema trackija ees.

MS Edge

Edge on täielikult Chrome’i kloon, ainult:

  • natuke rangem SmartScreen
  • natuke rohkem sandboxi
  • veidi piiranguid, kui “tracking protection” on High

Test ütleb:

  • FingerprintJS VisitorId ei muutu sessioonide vahel
  • ClientJS Fingerprint ei muutu sessioonide vahel → See tähendab: sama lihtsasti fingerprintitav kui Chrome.

Edge lisab:

  • Microsofti enda telemeetriat
  • Microsofti reklaamid
  • Bing integratsiooni

See muudab olukorra tegelikult halvemaks, kui soovid privaatsust.

Opera

Opera müüs end Hiina konsortsiumile ja:

  • Opera serveritega toimub pidev taustsünkroon
  • Opera VPN ei ole VPN, vaid proxy, mis käib läbi Vpn gateways (tihti trackitakse)
  • Anti-fingerprintingu mehhanismi pole olemas
  • Sinu CSV põhjal: jällegi igas sessioonis sama fingerprint

Opera näitab sama infot kui Chrome, lisaks:

  • seadme resolutsioon
  • fontide komplekt
  • audio fingerprint
  • canvas fingerprint
  • WebGL fingerprint
  • GPU nimi

Opera on pigem hullem kui Chrome, sest lisaks FP-le on neil oma "built-in analytics".

Chromium eksponeerib veebilehtedele:

  • täpne süsteemi fontide loend,
  • GPU/WebGL signatuurid,
  • brauseri ja OS-i täis-User-Agent,
  • ekraanisuuruse täpsed väärtused,
  • kõik WebRTC andmed,
  • pluginate ja codec-ide infot.

Chromiumi puhul ei toimu normaliseerimist ega randomiseerimist – iga väärtus, mis on sinu riistvara konfiguratsioonist saadav, jõuab trackeriteni. See on põhjus, miks Chrome ja temapõhised brauserid (Edge, Opera, Vivaldi jne) olid sinu testides ühed kõige stabiilsemad ja unikaalsemad fingerprintid – st brauser ei püüa sind mingil moel varjata.

Lisaks ei eemalda Chromium telemeetriat ega privaatsusandmete kogumist – brauser “lekib” peaaegu kõike, mida veebilehed võiksid soovida kasutada tuvastamiseks.

Seetõttu on Chromium-baasil brauserid anti-fingerprinting seisukohalt halvimad valikud.

Brave Browser – osaline ja üksnes minimaalne anti-fingerprinting

Brave põhineb Chromiumil, aga lisab oma “Shields” ja teatud anti-tracking kaitsed. Testid näitasid, et Brave’i fingerprintid olid sama profiili ja sessiooni puhul stabiilsed, isegi privaatses aknas. See tähendab, et Brave ei loo iga seansi jaoks erinevat identiteeti – brauser näeb välja sama tema fingerprinti genereerivate testide jaoks.

Brave’i eesmärk ei ole muuta iga kasutajat sama näoga massiks, vaid vähendada parameetrite hulka, mis võivad kergesti tuvastada sind kui unikaalset seadet. See toimub nii:

  • Brave blokeerib vaikimisi reklaamija ja kolmanda osapoole trackerite päringud;
  • brauser võib randomiseerida teatud väiksemaid API vastuseid, nt canvas/WebGL mõningaid signatuure, nii et mõningad skriptipõhised fingerprintid võivad erineda;
  • see blokeerib kolmanda osapoole küpsiseid ja teatud telemeetriat.

See kõlab hästi, aga tõsiasi on see, et Brave’i anti-fingerprinting ei standardiseeri ega ühenda kõiki Brave kasutajaid ühte suuresse “anonymity set’i”. See raskendab küll mõnda lihtsat fingerprinti (nt User-Agent), kuid sügavad parameetrid nagu GPU või fontide loend jäävad nähtavaks. Seetõttu võib Brave’i kasutaja olla küll vähem jälgitav, kuid pikaajaline fingerprint jääb püsima.

Brave kasutab ka liikuvaid või domeenipõhiseid muutusi, kus brauser võib iga domeeni külastuse korral varjata või muuta mõnda väärtust, kuid see ei taga universaalset anonüümsust. See on pigem anti-tracking, mitte anti-fingerprinting lahendus.

Firefox (tavaline) – keskpärane ja konfigureeritav

Firefox integreerib oma tuuma sisse mitmeid privaatsusmeetmeid, millest kõige tähtsam on privacy.resistFingerprinting seade (about:config). See muudab brauseri andmeid vähem eristuvaks. Firefoxi privaatsustehnoloogiad:

  • ajavööndi normaliseerimine;
  • ekraani mõõtmete “letterboxing”;
  • User-Agent’i üldistamine;
  • graafika-API vastuste moonutamine.

Mozilla sõnul need kaitsed vähendavad oluliselt fingerprinti unikaalsust ning toimivad koos Enhanced Tracking Protection ja RFP süsteemiga.

Firefoxi tugevus on see, et RFP-d saab käsitsi lubada, muutes brauseri sarnasemaks Tor Browserile. Miinuseks on see, et vaikimisi RFP pole täielikult lubatud ning olulised süsteemi detailid võivad endiselt lekkida.

LibreWolf – tugev Firefoxi fork

LibreWolf eemaldab telemeetria ja kõvasti karmistab Firefoxi privaatsusseadeid. See sisaldab:

  • telemeetria eemaldamist;
  • agressiivseid küpsisepoliitikaid ja trackerite blokeerimist;
  • resistFingerprinting kasutamist laiemalt;
  • uBlock Origin integreeritult.

LibreWolf piirab paljusid fingerprintingu andmepunkte, kuigi ei tee kasutajaid täielikult homogeenseks. Suurem osa identifikaatoreid on üldistatud või moonutatud.

Testide põhjal on LibreWolf märgatavalt parem kui Chrome/Edge, kuid mitte nii homogeenne kui Mullvad või Tor.

Mullvad Browser – põhimõtteliselt Tor Browser ilma Tor-võrguta

Mullvad Browser paistab testides silma, sest kasutab sama anti-fingerprinting tuuma kui Tor Browser, kuid ilma Tor võrguta. Olulised omadused:

  1. resistFingerprinting on vaikimisi sees ja laialdaselt rakendatud;
  2. User-Agent ja OS info on standardiseeritud;
  3. piiratud fontide komplekt, mitte süsteemsed fondid;
  4. mitmed API väärtused muutuvad iga seansiga;
  5. väga tugev anti-tracking konfiguratsioon uBlock Originiga;
  6. riistvara API-d on moonutatud või piiratud;
  7. ekraani mõõtmete letterboxing;
  8. kõik aknad on privaatsed.

Mullvad Browseri eesmärk on luua mass homogeneous set – kõik kasutajad näevad välja võimalikult identsed. Testid näitasid, et Mullvad Browseri fingerprintid kas olid identsed teiste kasutajatega või muutusid iga seansiga nii, et neid oli raske püsivalt seostada.

Seetõttu peetakse Mullvad Browserit üheks kõige tõhusamaks anti-fingerprinting brauseriks väljaspool Tor Browserit.
